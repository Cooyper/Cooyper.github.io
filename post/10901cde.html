路由组件的搭建

vue-router

  • components 文件夹：通常放置非路由组件（全局共用的组件）
  • views / pages 文件夹：通常放置路由组件

配置路由

  • 项目当中配置的路由一般放置在 router 文件中

总结

  • 路由组件与非路由组件的区别：
    • 路由组件一般放置在 views | pages 文件夹，非路由组件一般放置在 components 文件夹中
    • 路由组件一般需要在 router 文件夹中进行注册（使用的即为组件的名字），非路由组件在使用的时候，一般都是以标签的形式使用
  • 注册完路由，不管是路由组件，还是非路由组件，都会带有 $route 和 $router 属性
  • $route ：一般用于获取路由信息（路径、query、params等）
  • $router：一般进行编程式导航进行路由跳转（ push | replace ）

路由的跳转

  • 路由跳转有两种形式：
    • 声明式导航 router-link ，可以进行路由的跳转
    • 编程式跳转 push | replace，可以进行路由跳转
    • 注：声明式导航能完成的，编程式导航都能完成，但是编程式导航除了路由跳转，还可以做一些其他的业务逻辑

Footer 组件显示与隐藏

  • 显示或者隐藏组件：v-if | v-show
  1. 可以根据组件上的 $route 获取当前路由信息，通过路由路径判断 Footer 显示与隐藏
  2. 配置路由的时候，可以给路由添加路由元信息（meta），路由需要配置对象

路由传参

  • 路由跳转的几种方式：

    • 声明式导航 router-link （务必要有 to 属性），可以实现路由的跳转
    • 编程式导航：利用组件实例的 $router.push|replace 方法实现路由跳转（可添加自己需要的一些业务）。

  • 路由传参，参数书写的几种方式：

    1. 字符串形式

    2. 模板字符串

    3. 对象传参

    4. 一些面试题

      • 1.路由传递参数（对象写法）path 是否可以结合 params 参数一起使用？

      • 答：不可以。路由跳转的时候，对象的写法可以使 name、path 形式，但是需要注意的是，path 这种写法不能与 params 参数结合使用

      • 2.如何指定 params 参数可传可不传？

      • // router 注册组件部分
{
    name:'search',
    path:'/search/:value?',		// 在 params 参数后面加上 ? 即可指定 params 可传可不传
    component:Search,
    meta: {
        footerHide: true
    },
},

      • 4.路由组件能不能传递 props 数据

      • 答：可以。有三种方式：

      • {
    name:'search',
    path:'/search/:value?',	
    component:Search,
    meta: {
        footerHide: true
    },
    // 1.布尔值写法：params
    // props:true,
    // 2.对象写法：额外的给路由组件传递一些 props
    // props:{a:1,b:2},
    // 3.函数写法：可使用 params，query，通过 props 参数传递给路由组件
    props:($route)=>({keyword:$route.params.value,keyword:$route.query.keyword})
},
