TypeScript

特性

类型系统按照是否允许隐式类型转换来分类，分为强类型和弱类型。

Js、Ts 都是弱类型语言，Python 是强类型语言

元组 Tuple

Tuple 的作用就是限制元素的类型并且限制个数的数组，同时 Tuple 这个概念只存在于 TS，在 JS 上是不存在的。

对象 Object

函数 function

写法：function 和箭头函数 ()=>{}

定义函数通常会省略返回值类型，因为 TS 的类型推断能正确推断出返回值类型。如果写入返回值的类型，则必须要有对应类型的返回值。

参数类型：

  • 可选参数：配置可有可无的参数，通过 ? 实现，需要放到最后面
  • 默认参数：使用 = 实现（TS 会将添加了默认值的参数识别为可选参数）
  • 剩余参数：使用 ... 扩展运算符实现

特殊类型

any

any 类型是所有类型的顶级类型，任何类型都可以归于 any 类型。不指定变量类型则默认为 any 类型。

unknow

unkow 也是所有类型的顶级类型，但 unkonw 更加严格，主要体现在：1. unknow 会对值进行检测，即 unknow 只能赋值给 unknow 和 any 类型; 2. unknow 不允许定义的只有任何操作（如方法, new 等）。

Enum 枚举

数字枚举

  • 枚举类型默认为数字类型，默认从 0 开始以累加，如果有设置默认值，则只会对下面的值产生影响；
  • 支持反向映射（从成员值到成员们的映射）

字符串枚举

字符串枚举要注意的是必须要有默认值，不支持反向映射

联合类型

当 TS 不确定一个联合类型的变量到底是哪个类型的时候，只能访问此联合类型的所有类型里共有的属性或方法。

断言*

类型断言

在某些情况下，我们明确地知道某个值是什么类型，不需要 TS 判断，这时候就可以使用类型断言。**类型断言会告诉编译器，你不用给我检查，相信我，它就是这个类型。

类型断言使用的两种使用方式：

  • 尖括号
  • as（推荐）

非空断言

当类型检查器无法断定类型时，可以使用 ! 断言操作对象是非 null 和 undefined 类型。

使用非空断言后，编译器会默认我们只会传来 string 类型的数据，所以可以赋值给 str1

双重断言

在断言失效的情况下，可以使用双重断言，但一般情况不用。

类型守卫

类型守卫是可执行运行时检查的一种表达，用于确保该类型在一定的范围内。常用守卫类型主要有以下四种：

in 关键字

typeof 关键字

用于判断基本类型，例如 string | number 等

instanceof 关键字

用于判断一个实例是不是某个构造函数的实例

断言与类型守卫的概念非常相似，都是确定参数的类型。断言是直接告诉编译器某个参数就是这个类型，类型守卫更像确定这个参数具体是什么类型。

类型断言的使用限制：

  • 联合类型可以被断言为其中一个类型
  • 父类可以被断言为子类
  • 任何类型都可以被断言为 any
  • any 可以被断言为任何类型
  • 要使得 A 能够被断言为 B，只需要 A 兼容 B 或 B 兼容 A 即可

注意：类型断言不是类型转换，它不会真的影响到变量的类型。

类型别名和接口*

类型别名 type

类型别名用于给某个类型起个新名字

接口 interface

接口 ( Interfaces ) 一般用于定义对象的类型。它可以包括以下数据：可选属性、只读属性、任意属性。

  • 可读属性：当我们定义一个接口时，我们的属性可能不需要全都要，这是就需要 ? 来解决
  • 只读属性：用 readonly 修饰的属性为只读属性，意思是指允许定义，不允许之后进行更改
  • 任意属性：这个属性极为重要，它是可以用作就算没有定义，也可以使用，比如 [data: string]: any。比如说我们对组件进行封装，而封装的那个组件并没有导出对应的类型，然而又想让他不报错，这时就可以使用任意属性。

继承

和类一样，接口也可以继承属性，通过 extends 字段实现

函数类型接口

除了对象，interface 还可以定义函数。

type 和 interface 的区别

type 和 interface 非常相似，在大多数情况下是等价的，但在一些特定场景下是有挺大区别的。

基础数据类型

  • type 和 interface 都可以定义对象和函数
  • type 和可以定义其他数据类型，如字符串、数字、元组、联合类型等，而 interface 不行

扩展

interface 可以扩展 type，type 也可以扩展为 interface，但两者实现扩展的方式不同。

  • interface 是通过 extends 来实现
  • type 是通过 & 来实现

重复定义

interface 可以多次被定义，并且会进行合并，但 type 不行

泛型**

泛型：Generics，是指在定义函数、接口或类的时候，不预先指定具体的类型，而在使用的时候再指定类型的一种特性。

也就是说，泛型是允许同一个函数接受不同类型参数的一种模版，与 any 相比，使用泛型来创建可复用的组件要更合适，因为泛型会保留参数类型。

泛型常用字母

用常用的字母来表示一些变量的代表：

  • T：代表Type，定义泛型时通常用作第一个类型变量名称

声明文件*

声明语句

声明文件

通常声明语句会放到一个单独的文件 xxx.d.ts 中，该文件就是声明文件。声明文件必须以 .d.ts 为后缀。TS 会解析所有的 *.ts 文件，包括 *.d.ts 文件。所以，在 *.d.ts 文件中声明了类型，在其他所有 *.ts 文件中可以获得类型定义并进行使用。

声明文件书写

全局变量：

  • declare var/let/const 声明全局变量

  • declare function 声明全局方法

  • declare class 声明全局类

  • declare enum 声明全局枚举类型

  • declare namespace 声明（含有子属性的）全局对象

  • declare interface 和 declare type 声明全局类型

常用技巧

Record

Record 语法：Record<K extends keyof any, T>

作用：将 K 中所有的属性的值转化为 T 类型。

Pick

Pick 语法：Pick<T, K extends keyof T>

作用：将某个类型中的子属性挑出来，变成包含这个类型部分属性的子类型。

Omit

Omit 语法：Omit<T, U>

作用：将已经声明的类型进行属性剔除获得新类型。

Omit 与 **Exclude **的区别：Omit 返回的是新的类型，原理上是在 Exclude 之上进行的，Exclude 是根据自类型返回的。

Exclude

Exclude 语法：Exclude<T, U>

作用：将 T 类型中的 U 类型剔除。
