剑指 Offer 12. 矩阵中的路径

题目

难度：中等

给定一个 m x n 二维字符网格 board 和一个字符串单词 word 。如果 word 存在于网格中，返回 true ；否则，返回 false 。

单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母不允许被重复使用。

提示：

  • 1 <= board.length <= 200
  • 1 <= board[i].length <= 200
  • board 和 word 仅由大小写英文字母组成

题解

该题是典型的矩阵搜索问题，可考虑使用 深度优先遍历（DFS） + 剪枝 的方法。

  • 深度优先搜索：暴力遍历矩阵中所有的字符串可能性。DFS 通过递归，先朝一个方向遍历到底，再回溯至上个节点，沿另一个方向搜索，以此类推。
  • 剪枝：在搜索过程中，遇到 这条路不可行 和 目标字符串匹配成功 的情况（例如：此矩阵元素和目标字符不同、此元素已被访问），则立即返回，称之为 可行性剪枝。

DFS 解析

  • 递归参数：当前元素在矩阵 board 中的行列索引 i 和 j ，当前目标字符在 word 中的索引 k
  • 终止条件：
    1. 返回 false ：
      1. 下标越界
      2. 当前元素 board[i][j] 不等于 word[k]
      3. 当前元素 board[i][j] 已被访问（可与2合并）
    2. 返回 true ：k == word.size() - 1，即 word 已全部匹配。
  • 递推工作：
    1. 标记当前元素 board[i][j] 已被访问，即 board[i][j] = '\0' ，防止后续的 DFS 重复访问
    2. 递归访问当前元素的上右下左方向的元素，使用 或 条件来连接，即找到一条路径就停止当前递归，不再做后续的 DFS，将搜索结果记录到 hasPath。
    3. 还原当前矩阵元素：将 board[i][j] 还原至原始值，即 word[k]
  • 返回值：返回结果布尔变量 hasPath，代表是否找到路径

复杂度分析

  • 时间复杂度：O(3K MN) 。每次选择有三个方向，最坏情况需要选择 K 次，即 3K 次。又因为一共有 M * N 个起点，故时间复杂度为 O(3K MN) 。
  • 空就复杂度：O(K)。调用递归的过程中深度不超过 K ，所以调用函数累积使用的栈空间占用 O(K)，最坏的情况为 K = M * N，即此时空间复杂度为 O(K)。
